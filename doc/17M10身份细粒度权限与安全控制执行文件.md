# M10 身份、细粒度权限与安全控制执行文件

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development or superpowers:executing-plans. Security behavior must be test-first.

**Goal:** 从角色级工具权限升级到具备身份上下文、部门/租户隔离、行字段权限、脱敏、Prompt Injection 防护和 HITL 的安全执行层。

**Architecture:** `SecurityContext` 由认证层创建，Permission Engine 在 Planner、Skill 和工具调用边界统一执行策略。业务查询服务接收数据范围，不允许工具自行扩大权限。高风险操作通过 LangGraph interrupt/checkpoint 暂停等待审批。

**Tech Stack:** pydantic, python-jose, cryptography, LangGraph interrupts, pytest。

## 2. 安全上下文

```python
SecurityContext(
    tenant_id,
    user_id,
    roles,
    department_id,
    permissions,
    data_scopes,
    auth_source,
)
```

Runtime 内部不再信任 CLI/API 直接传入的 role 字符串；本地开发可使用显式 `DEV_AUTH_ENABLED` 构造测试身份。

## 3. 执行任务

### Task 1：Identity 与认证边界

- [ ] 测试 JWT 有效、过期、签名错误、tenant 缺失。
- [ ] API/CLI 将身份解析为 `SecurityContext`。
- [ ] Trace 只记录身份 ID 和权限摘要，不记录 Token。

### Task 2：Permission Engine

- [ ] 策略输入：

```text
security_context
tool/skill
action
risk_level
resource
```

- [ ] 输出：

```text
allow
deny
require_approval
```

- [ ] 未明确允许默认拒绝高风险操作。

### Task 3：Tenant、部门和行级数据范围

- [ ] `query_sql` 接收强制 scope；
- [ ] manager 只能查授权部门；
- [ ] admin 也必须受 tenant 限制；
- [ ] 用户输入不能覆盖 scope；
- [ ] SQL 测试验证跨 tenant/department 数据不会返回。

### Task 4：字段脱敏

- [ ] 定义字段策略：

```text
public
internal
sensitive
restricted
```

- [ ] 未授权角色对账户、个人信息等字段脱敏；
- [ ] 脱敏发生在工具结果进入 LLM Context 和 Trace 之前。

### Task 5：Prompt Injection 与工具注入防护

- [ ] 检测文档中的：

```text
忽略系统指令
泄露密钥
绕过权限
调用未授权工具
```

- [ ] 检索文档始终标记为不可信数据，不作为系统指令解释。
- [ ] 模型输出的工具名和参数必须通过 schema、白名单和 Permission。

### Task 6：HITL

- [ ] 高风险操作返回 `require_approval`；
- [ ] 使用 Checkpoint 保存待审批状态；
- [ ] 支持 approve/reject/timeout；
- [ ] 恢复时重新检查身份和权限；
- [ ] 审批不能修改原始工具参数而不生成新审计记录。

### Task 7：Audit

- [ ] 审计记录：

```text
who
tenant
action
resource
decision
policy
request_id
timestamp
```

- [ ] 审计日志与普通 Trace 分离；
- [ ] 敏感输入只保存摘要或 hash。

### Task 8：安全 Eval

- [ ] 新增至少 30 条安全任务：
  - 跨用户会话；
  - 跨部门查询；
  - 跨 tenant 查询；
  - prompt injection；
  - MCP 未授权调用；
  - Skill 越权；
  - HITL。
- [ ] 指标：

```text
Unauthorized Block Rate
Cross-tenant Leakage Rate
Redaction Accuracy
Injection Blocking Rate
HITL Decision Accuracy
```

## 4. 验收标准

- 未认证请求无法进入 Runtime；
- tenant/user/thread 形成完整隔离；
- 数据范围由服务端策略注入；
- 敏感字段进入 LLM 前已脱敏；
- 文档指令不能改变系统工具权限；
- 高风险操作可暂停、审批和恢复；
- 安全 Eval 不出现跨 tenant 泄漏；
- M1-M9 测试继续通过。
