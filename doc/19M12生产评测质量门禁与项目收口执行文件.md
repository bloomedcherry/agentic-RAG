# M12 生产评测、质量门禁与项目收口执行文件

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development or superpowers:executing-plans. Report only measured values.

**Goal:** 将 Demo Eval 升级为覆盖 Memory、LLM、Skill、MCP、RAG 和安全的生产回归体系，并在 CI 中建立可执行的质量门禁。

**Architecture:** Eval task 使用版本化 Gold Set；每次运行生成独立 run_id 和结果目录。离线指标、性能指标、安全指标和成本指标分别计算，最终由 gate 脚本依据明确阈值决定通过或失败。

**Tech Stack:** pytest, JSONL, Markdown/JSON reports, GitHub Actions, statistics。

## 3. 执行任务

### Task 1：版本化 Gold Set

- [ ] manifest 记录版本、审核人、日期和 schema；
- [ ] 校验重复 ID、无效 source、未知工具/Skill；
- [ ] Gold Set 变更必须在 PR 中可审阅。

### Task 2：独立 Eval Run

- [ ] 每次运行生成：

```text
run_id
config snapshot
model names
index version
skill versions
git commit
metrics.json
failures.jsonl
summary.md
```

- [ ] 不再依赖混有历史记录的单一 traces.jsonl 计算整轮指标。

### Task 3：Memory Eval

- [ ] 指标：

```text
Context Recall
Reference Resolution Accuracy
Session Isolation Accuracy
Restart Recovery Rate
Summary Consistency
```

### Task 4：Skill Eval

- [ ] 指标：

```text
Skill Selection Accuracy
No-skill Fallback Accuracy
Skill Completion Rate
Progressive Loading Compliance
```

### Task 5：MCP Eval

- [ ] 指标：

```text
MCP Tool Selection Accuracy
MCP Success Rate
Permission-before-network Accuracy
Timeout Recovery Rate
MCP P95 Latency
```

### Task 6：RAG 与答案质量

- [ ] 指标：

```text
Recall@1/3/5
MRR
Citation Accuracy
Citation Support Accuracy
Answer Point Coverage
```

- [ ] Citation Support 不能只检查字符串出现，至少包含规则审查集；LLM-as-Judge 可选但不能作为唯一真值。

### Task 7：安全 Eval

- [ ] 指标：

```text
Unauthorized Block Rate
Cross-tenant Leakage Rate
Prompt Injection Blocking Rate
Redaction Accuracy
HITL Accuracy
```

- [ ] 任意跨 tenant 泄漏直接门禁失败。

### Task 8：延迟、稳定性和成本

- [ ] 统计：

```text
Average / P50 / P95 / P99 latency
LLM tokens
estimated cost
retry count
fallback rate
error distribution
```

- [ ] 模型费用单价来自版本化配置，不写死在代码。

### Task 9：Baseline 与消融

- [ ] 比较：

```text
规则模板 baseline
+ LLM
+ Memory
+ Skill
+ MCP
+ Hybrid RAG
Full system
```

- [ ] 每个版本使用相同 Gold Set 和配置快照。

### Task 10：Quality Gate

- [ ] 初始门禁：

```text
M1-M12 tests: 100% pass
Tool Call Accuracy >= 0.95
Permission Blocking Accuracy = 1.00
Session Isolation Accuracy = 1.00
Cross-tenant Leakage Rate = 0
Citation Accuracy >= 0.95
Hybrid RAG Recall@5 > 0.10
```

- [ ] 延迟与成本先作为报告项，获得稳定基线后再设硬门槛。
- [ ] 门禁失败输出具体 case IDs。

### Task 11：CI

- [ ] PR 运行：
  - 单元测试；
  - Fake LLM/MCP 离线评测；
  - 安全门禁；
  - 配置和 Skill schema 校验。
- [ ] 真实模型全量评测使用手动或定时 workflow，Secret 不进入日志。

### Task 12：项目收口

- [ ] 更新 README、架构图、运行手册、故障处理手册；
- [ ] 更新 `doc/11工业生产级Agent差距分析.md` 的完成状态；
- [ ] 输出最终项目复盘和简历表达；
- [ ] 不写未经对照实验验证的提升百分比。

## 5. 验收标准

- Eval Set 有版本和人工审核信息；
- 每轮评测结果独立、可复现；
- Memory、Skill、MCP、安全均有指标；
- CI 离线门禁不依赖外部模型；
- 真实模型评测可手动运行；
- 安全硬门禁生效；
- 报告包含失败案例、延迟、Token 和成本；
- 项目文档只引用实测数据。
